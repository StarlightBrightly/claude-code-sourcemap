# Claude Code 命令系统详解

## 命令系统架构

命令系统提供用户可调用的交互式命令,通过 `/` 前缀触发。

## 命令注册

所有命令在 `commands.ts` 中注册:

```typescript
import commit from './commands/commit.js'
import review from './commands/review.js'
import config from './commands/config/index.js'

export function getCommands(): Command[] {
  return [
    commit,
    review,
    config,
    // ...
  ]
}
```

## 核心命令分类

### 1. Git 相关命令

**commit** - Git 提交
- 自动暂存更改
- 生成提交消息
- 执行提交

**review** - 代码审查
- 分析代码变更
- 提供改进建议
- 检查最佳实践

**commit-push-pr** - 提交并创建 PR
- 提交更改
- 推送到远程
- 创建 Pull Request

### 2. 配置命令

**config** - 配置管理
- 查看配置
- 修改配置
- 重置配置

**theme** - 主题设置
- 切换主题
- 自定义颜色

**vim** - Vim 模式
- 启用/禁用 Vim 模式
- 配置快捷键

### 3. 会话管理命令

**resume** - 恢复会话
- 列出历史会话
- 恢复指定会话

**session** - 会话操作
- 查看当前会话
- 切换会话
- 删除会话

**clear** - 清除会话
- 清除消息历史
- 保持上下文

### 4. MCP 命令

**mcp** - MCP 管理
- 列出 MCP 服务器
- 添加/删除服务器
- 测试连接

子命令:
- `mcp add`: 添加 MCP 服务器
- `mcp list`: 列出服务器
- `mcp remove`: 删除服务器
- `mcp test`: 测试连接

### 5. 任务管理命令

**tasks** - 任务管理
- 创建任务
- 更新任务状态
- 列出任务

### 6. 帮助与文档命令

**help** - 帮助信息
- 显示命令列表
- 显示命令详情

**doctor** - 诊断工具
- 检查环境
- 诊断问题
- 提供修复建议

### 7. 认证命令

**login** - 登录
- OAuth 认证
- API 密钥设置

**logout** - 登出
- 清除认证信息

### 8. 分析命令

**cost** - 成本分析
- 显示 Token 使用
- 估算成本

**usage** - 使用统计
- 显示使用情况
- 统计信息

**stats** - 统计信息
- 会话统计
- 工具使用统计

### 9. 开发工具命令

**diff** - 查看差异
- Git diff
- 文件对比

**context** - 上下文管理
- 查看当前上下文
- 添加/移除文件

**memory** - 内存管理
- 查看内存
- 添加/删除记忆

### 10. 高级命令

**teleport** - 远程会话
- 传送到远程环境
- 远程执行

**bridge** - 桥接模式
- IDE 集成
- 远程控制

**voice** - 语音模式
- 语音输入
- 语音输出

## 命令实现模式

### 1. 简单命令

```typescript
export default {
  name: 'clear',
  description: 'Clear the conversation',
  action: async () => {
    // 实现逻辑
  }
}
```

### 2. 带参数命令

```typescript
export default {
  name: 'resume',
  description: 'Resume a session',
  options: [
    { name: 'session-id', type: 'string' }
  ],
  action: async (options) => {
    const sessionId = options.sessionId
    // 实现逻辑
  }
}
```

### 3. 子命令

```typescript
export default {
  name: 'mcp',
  description: 'MCP management',
  subcommands: [
    {
      name: 'add',
      action: async () => { /* ... */ }
    },
    {
      name: 'list',
      action: async () => { /* ... */ }
    }
  ]
}
```

## 命令执行流程

```
用户输入 → 命令解析 → 参数验证 → 执行 → 结果显示
```

## 条件编译命令

某些命令仅在特定条件下可用:

```typescript
const voiceCommand = feature('VOICE_MODE')
  ? require('./commands/voice/index.js').default
  : null
```

## 命令权限

命令可以有权限要求:
- 需要认证
- 需要 Git 仓库
- 需要特定配置
