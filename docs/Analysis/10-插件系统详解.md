# Claude Code 插件系统详解

## 插件架构

插件系统允许扩展 Claude Code 的功能。

## 插件结构

### 1. 插件定义

```typescript
interface Plugin {
  name: string
  version: string
  description: string

  // 生命周期钩子
  onLoad?: () => Promise<void>
  onUnload?: () => Promise<void>

  // 扩展点
  tools?: Tool[]
  commands?: Command[]
  skills?: Skill[]
}
```

### 2. 插件目录

- 内置插件: `src/plugins/bundled/`
- 用户插件: `~/.claude/plugins/`
- 项目插件: `.claude/plugins/`

## 插件加载

### 1. 加载流程

```
扫描插件目录 → 解析 package.json → 加载代码 → 初始化 → 注册
```

### 2. 版本管理

```typescript
// 插件版本
{
  "name": "my-plugin",
  "version": "1.0.0",
  "claudeCode": "^2.0.0"
}
```

### 3. 依赖管理

插件可以声明依赖:

```json
{
  "dependencies": {
    "other-plugin": "^1.0.0"
  }
}
```

## 插件 API

### 1. 注册工具

```typescript
export function registerTools(registry: ToolRegistry) {
  registry.register(new MyTool())
}
```

### 2. 注册命令

```typescript
export function registerCommands(registry: CommandRegistry) {
  registry.register({
    name: 'my-command',
    action: async () => { /* ... */ }
  })
}
```

### 3. 注册技能

```typescript
export function registerSkills(registry: SkillRegistry) {
  registry.register({
    name: 'my-skill',
    prompt: '...'
  })
}
```

## 插件隔离

### 1. 沙箱环境

插件在隔离环境中运行:
- 独立的全局作用域
- 受限的文件系统访问
- 受限的网络访问

### 2. 权限系统

插件需要声明权限:

```json
{
  "permissions": [
    "filesystem:read",
    "network:fetch",
    "tools:register"
  ]
}
```

## 内置插件

### 1. 核心插件

- `git-integration`: Git 集成
- `github-integration`: GitHub 集成
- `code-analysis`: 代码分析

### 2. 插件管理

```bash
# 列出插件
/plugin list

# 安装插件
/plugin install <name>

# 卸载插件
/plugin uninstall <name>
```
